我的第一篇博客文章

[[如何用Nas和Obsidian搭建个人博客]] 好的，收到！一份清晰、可执行的项目任务清单，是把宏大想法变为现实的第一步。 [[提升效率]] 个人博客（漂流瓶计划）搭建任务清单 [!note] 项目目标 将Obsidian中的思考与沉淀，通过一个完全由我掌控的个人博客发布出去。它是一个自我记录的平台，也是一个期待回响的漂流瓶。 Phase 0: 奠基与准备 (Foundation & Prep) 思想准备： 在Obsidian中建立一个文件夹，例如 _PUBLISH 或 _BlogPosts，专门存放计划发布的文章。 内容准备： 挑选 3-5 篇已有的、你最想分享的笔记作为首批发布的“漂流瓶”。 技术约定： 在这几篇笔记的顶部，添加并配置好 YAML Frontmatter，至少包含 title, date, tags 和 publish: true。 yaml --- title: "我的第一篇博客文章" date: 2023-10-27 tags: [随笔, 技术] publish: true --- Phase 1: 本地“船坞”建设 (Local Development) 目标：在你的电脑上成功运行一个博客的本地预览版。 安装引擎： 在你的5090台式机上，根据 Hugo官方文档 安装 Hugo。 创建站点： 使用命令行 hugo new site MyBlog 创建一个新的博客项目文件夹。 选择主题： 浏览 Hugo Themes，选择一个你喜欢且支持双链或易于定制的主题（如 Quartz, Coder, MemE）。 分叉而不是下载 (Fork it): 去 Quartz的GitHub页面，点击右上角的 “Fork” 按钮。这会把整个项目完整地复制一份到你自己的GitHub账户下。现在，这个项目完全属于你了，你可以随心所欲地修改，而不用担心影响原作者。 克隆到本地： 使用 git clone 命令，将你自己的那个分叉项目，克隆到你的电脑上。 跑起来： 按照Quartz的官方文档，完成初始化安装，让它能在你的本地成功运行起来。这是你探索的起点。 安装主题： 按照主题文档的指引，将其安装到你的 MyBlog 项目中。 也可以用别的主题： Amethyst 核心配置： 编辑项目根目录下的 hugo.toml (或 config.toml) 文件，至少配置好 baseURL, languageCode, title, 和 theme。 内容同步： 将你Obsidian中的 _PUBLISH 文件夹软链接 (Symbolic Link) 到Hugo项目的 content 目录下。 本地验证： 在 MyBlog 项目根目录下运行命令 hugo server，然后在浏览器打开 http://localhost:1313，确认能看到你的博客和文章。 实战演习：排除重大部署故障 笔记：Hugo“幽灵Bug”狩猎日志 背景介绍（给朋友看版）： ...
