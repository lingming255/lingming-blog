Phase 1: 本地“精装样板间”建设 (Local Development)#

目标：在你的电脑上，使用新主题构建一个功能完善、设计满意的本地预览版。

  • 安装引擎： 在你的台式机上安装 Hugo。

  • 创建站点： 使用 hugo new site MyBlog 创建项目。

  • 安装新主题 (PaperMod)：

    • 进入项目目录： 打开终端，cd MyBlog。

    • 初始化Git仓库： 运行 git init，这是使用子模块的前提。

    • 安装主题为Git子模块： 运行以下命令，将PaperMod作为可独立更新的模块添加到你的项目中。这是最现代化的主题管理方式。

      git submodule add https://github.com/adityatelange/hugo-paper.git themes/papermod

  • 核心配置 (hugo.toml)： 编辑项目根目录下的 hugo.toml 文件，应用新主题并进行基础配置。

    baseURL = "https://lingming.art/" # 最终要部署的域名languageCode = "zh-cn" # 语言title = "lingming的漂流瓶"theme = "papermod"[params]  # 启用搜索功能  ShowPostNavLinks = true  ShowBreadCrumbs = true  ShowReadingTime = true  ShowShareButtons = true  ShowCodeCopyButtons = true    [params.homeInfoParams]  Title = "你好，我是lingming"  Content = "this is Self-Introduction"  [params.assets]  # 禁用主题自带的jquery，如果需要可开启  disableFingerprinting = true[menu]  [[menu.main]]    identifier = "archives"    name = "归档"    url = "/archives/"    weight = 10  [[menu.main]]    identifier = "search"    name = "搜索"    url = "/search/"    weight = 20  [[menu.main]]    identifier = "tags"    name = "标签"    url = "/tags/"    weight = 30

  • 内容同步： 保持用软链接方式将Obsidian发布目录同步到content。

  • 本地验证： 在 MyBlog 根目录下运行 hugo server，在浏览器打开 http://localhost:1313，确认新主题 PaperMod 的外观和功能符合预期。

Phase 2: “云端军火库”与“自动化兵工厂”对接 (Git & Vercel)#

目标：将本地的“精装图纸”上传到GitHub，并授权Vercel进行全自动部署。

  • 建立“忽略规则” (.gitignore)：

    • 在项目根目录创建 .gitignore 文件，写入以下内容，防止将临时文件和最终成品上传到源码仓库。

  • 连接GitHub“云端军火库”：

    • 创建GitHub私有仓库： 已完成。

    • 将本地项目与远程仓库关联：

      # 注意替换为你的仓库地址git remote add origin https://github.com/YourName/your-repo.git

  • 首次提交“设计蓝图”：

    • 运行以下命令，将你的整个Hugo项目（作为设计蓝图）推送到GitHub。

      git add .git commit -m "Initial commit with PaperMod theme and base configuration"git push -u origin main # 或者 master

  • 连接Vercel“自动化兵工厂”：

    • 使用你的GitHub账号登录Vercel。
    • 选择 Import Project，并从列表中选择你刚刚推送的GitHub仓库。
    • Vercel会自动识别出这是一个Hugo项目，你无需修改任何构建设置。
    • 点击 Deploy。等待几十秒，Vercel会给你一个 *.vercel.app 的域名，你的网站就已经全球上线了！

  • 绑定你的专属域名：

    • 在Vercel的项目设置中，进入 Domains 选项卡。
    • 输入你的域名 lingming.art 并添加。
    • Vercel会引导你到你的域名提供商（如GoDaddy, Namecheap）后台，修改NS记录或CNAME记录。按照Vercel的指引操作即可。

Phase 3: 功能增强与灵魂注入 (Feature Enhancement)#

目标：为你的PaperMod主题博客添加你期望的高级功能。

  • 部署评论系统 (Giscus)：

    • Giscus 是一个利用GitHub Discussions的免费、无后台、高逼格的评论系统。

    • 按照 Giscus 官网的指引，在你的博客GitHub仓库中启用Discussions并完成配置。

    • 将获取到的配置信息，填入你本地的 hugo.toml 文件中：

      [params.giscus]  repo = "YourName/your-repo" # 你的仓库  repoId = "..." # Giscus提供的仓库ID  category = "Announcements" # 你选择的Discussion分类  categoryId = "..." # 分类ID  mapping = "pathname"  strict = "0"  reactionsEnabled = "1"  emitMetadata = "0"  inputPosition = "top"  theme = "preferred_color_scheme"  lang = "zh-CN"  loading = "lazy"

  • 实现反向链接 (Backlinks)：

    • PaperMod 本身不直接支持，但我们可以通过修改模板添加。这是一个进阶任务，我们可以后续攻克。目前的重点是让核心系统运转起来。

  • 注入“动态标签页标题”灵魂：

    • 创建JS文件： 在你的项目 assets/js/ 目录下（如果不存在就新建），创建一个名为 dynamic_title.js 的文件。

    • 粘贴代码： 将以下代码粘贴到 dynamic_title.js 文件中。

      (function() {    const originalTitle = document.title;    const awayMessage = "别走嘛QAQ";    const welcomeMessage = "欢迎QWQ";    document.addEventListener('visibilitychange', function() {        if (document.visibilityState === 'hidden') {            document.title = awayMessage;        } else {            document.title = welcomeMessage;            setTimeout(function() {                document.title = originalTitle;            }, 2000);        }    });})();

    • 在hugo.toml中引入脚本：

      [params]  # ... 其他params配置  customJS = ["js/dynamic_title.js"]

Phase 4: 巡航与维护 (The New Workflow & Maintenance)#

目标：享受全新的创作流程，并定期维护你的系统。

  • 你的全新工作流：
      1. 在Obsidian中创作。
      1. 在终端中运行 git push。
      1. 结束。 Vercel会在几十秒内自动完成所有后续工作。
  • 维护计划：
    • 主题更新： 定期运行 git submodule update --remote --merge 来更新PaperMod主题到最新版。
    • 引擎更新： 如果需要，可以在Vercel的项目设置中，修改HUGO_VERSION环境变量来升级Hugo引擎。

  • 第一重故障：致命的“幽灵Bug”
    • 现象： 全量笔记部署导致服务器崩溃。
    • 定位方法：【二分查找法】 指数级效率锁定单一故障文件：《技术问题.md》。
    • 病因分析： 文件缺少 Front Matter，导致Hugo引擎核心恐慌。
  • 第二重故障：“隐形”的文件夹层级
    • 现象： 深层文件夹无法在导航栏显示。
    • 病因分析： 主题配置的导航范围(BookSection)受限，且目标文件夹缺少_index.md“名片”。
  • 第三重故障：无法折叠的顶级菜单
    • 现象： 顶级菜单项无法折叠。
    • 病因分析： 内容结构与主题设计哲学不匹配，需调整内容层级以适应主题。
  • 战役核心收获：定位问题的三个层次
    1. 逻辑层定位（二分法）： 系统性排除。
    2. 规则层定位（查阅默认配置）： 寻找“权威”信息。
    3. 架构层定位（理解设计哲学）： “修复”不够时，要学会“适应”。
